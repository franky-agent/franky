### What was implemented

- **`src/coding/restart.zig`** — new module with `init`, `deinit`,
  `spawnAndExit`. Caches exe path + argv at bootstrap and spawns a
  fresh process via `std.process.spawn` then calls `std.process.exit(0)`.
- **`src/coding/mod.zig`** — exports `pub const restart`.
- **`Session` struct** — added `restart_requested: std.atomic.Value(bool)`.
- **`proxy.run()`** — accepts `original_argv` parameter; passes it to
  `initSession`.
- **`initSession()`** — calls `restart_mod.init(original_argv, ...)` to
  cache argv. Best-effort; logs a warning if caching fails.
- **Accept loop** — polls `restart_requested` flag on each iteration;
  breaks out when set.
- **Restart sequence** (after loop break) — closes the listen socket
  (`server.deinit`), broadcasts `event: close\ndata: {}\n\n` to SSE
  subscribers, calls `restart_mod.spawnAndExit`. If spawn fails (OOM),
  logs an error and returns (process exits cleanly since socket is gone).
- **`/restart` slash command** — registered in `buildProxySlashRegistry`.
  Sets `side_effect = .restarting` and `restart_requested = true`.
- **`POST /restart` HTTP route** — set the restart flag and respond
  `{"ok":true,"restarting":true}`. The main accept loop picks it up.
- **Unit test** for `init`/`deinit` caching logic.
- **`/restart` slash command test** — verifies `.restarting` side-effect,
  `restart_requested` flag, and output message.
- **`POST /restart` HTTP route test** — verifies HTTP 200,
  `{"ok":true,"restarting":true}` response, and flag via
  `runProxyHttpRequest`.
- **Help text** — `/restart` listed in `/help` output + covered by
  the help-list test.

## Context: what already works

- **Binary replacement** (`src/coding/update.zig`): `replaceExecutable` writes new
  bytes to `<exe>.tmp`, renames atomically over the running binary. The on-disk
  binary is swapped. But the *running process* still executes the old text segment
  — the swap has no effect until the process exits and restarts.
- **Session persistence**: `persistSession()` writes transcript to
  `~/.franky/sessions/<id>/`. On reconnect, the replay ring + `GET /transcript`
  rehydrate the browser. **But the replay ring is in-memory** — it's lost on restart.
- **Proxy listen loop** (`src/coding/modes/proxy.zig:128`): `while (true) { accept; spawn thread; detach }`.
  No restart mechanism existed.
- **Socket options**: `reuse_address = true` on the listen socket — a new process
  can bind the same port immediately after the old one releases it.

---

## Approach B — spawn-and-exit (fork + exec, parent dies) ★ chosen & implemented

Current process spawns a child with `std.process.spawn(io, .{ .argv = … })`,
gives it a short head start, then exits normally. The child starts `main()` fresh.

```
trigger ──► server.deinit(io)    // close listen socket FIRST → port freed
        ──► spawn(new_binary, argv, env)
        ──► sleep(200ms)          // brief head start for child
        ──► exit(0)               // normal exit, defer chains run
                                    └── child binds port, initializes
```

**Port-conflict mitigation (key detail):** The parent calls `server.deinit(io)`
*before* spawning the child. By the time the child starts `main()` and binds,
the port is free. The proxy already sets `reuse_address = true` on the listen
socket, so even if the parent's socket lingers in `TIME_WAIT` the child's bind
won't get `EADDRINUSE`. The gap between `server.deinit` and the child's `bind`
is typically <1ms — negligible for a localhost-facing service.

**Pros:**
- Simplest to implement (no fd-passing, no execve(2) edge cases)
- `std.process.spawn` already exists in the codebase (`bin/franky_profile.zig`)
- No signal handlers needed — can be triggered from a slash handler or HTTP route
- Parent exits *normally* → `defer` chains fire, destructors run, state is
  cleaned up by the existing code. No gotcha with skipped deferreds.
- Works on macOS (same `std.process.spawn` API, same POSIX primitives)
- **Complexity: 1.5/5**

**Cons:**
- Two processes briefly coexist (the parent exiting while the child boots)
- Child inherits the parent's fd table; parent should close non-inheritable
  fds before spawning (already handled by `server.deinit`)
- ~200ms port-free window after `server.deinit` (in practice <1ms on a
  local network). New connections during that window get `ECONNREFUSED`. The
  browser's EventSource retries automatically.

---

## Platform compatibility: both A and B work on macOS

Both `execve` (Approach A) and `std.process.spawn` (Approach B) are POSIX
primitives available on macOS through libSystem. The proxy already links libc
(`link_libc = true` in `build.zig`), so both paths are available on both
platforms with zero changes.

## Recommended path: Approach B (spawn-and-exit) for v1

Rationale: it's the simplest correct path. The parent exits normally through
the existing `defer session.deinit()` chain, which already persists the
transcript, frees memory, and cleans up threads. No `execve`-never-returns
gotchas, no fd coordination, no signal-handler safety concerns.

The one con — brief port gap — is mitigated by closing the listen socket
*before* spawning (`server.deinit(io)` first). The gap is sub-ms on localhost
and browsers retry EventSource connections automatically.

### Trigger surface

- **Implemented:** `/restart` slash command + `POST /restart` HTTP route.
- `finish_task` extension (adding a `restart` bool field) is deferred to v2.
- SIGHUP signal handling is deferred to v2 (adds signal-safety concerns).

### v1 scope

- **Proxy mode only.** `--mode proxy` is the primary target. Print mode is
  single-shot (no process to restart after it returns). Interactive mode gets
  it in v2.

### How it works, end-to-end

```
1. User (or agent calling finish_task) triggers restart
2. POST /restart handler fires (or /restart slash command)
3. Handler sets session.restart_requested = true
4. Server closes the listen socket: server.deinit(io)  ← port freed
5. Server spawns child with argv preserved from bootstrap
6. Server sleeps 200ms (brief head start for child)
7. Child's main() runs, binds port (reuse_address = true → no EADDRINUSE)
8. Parent exits cleanly through defer chains → session persists, replay
   ring is lost but transcript is on disk
9. Browser's EventSource sees connection drop → reconnects with
   Last-Event-ID → new process has empty ring → sends replay_gap →
   browser falls back to GET /transcript → full rehydration
```

### Argv preservation

In proxy mode, the original `argv` is available in `print_mode.run()` at the
call site where `proxy.run()` is invoked. We store it on `Session` as a
duplicated `[]const []const u8`:

```zig
// On Session:
restart_argv: ?[]const []const u8 = null,
```

Set once in `initSession()`, freed in `Session.deinit()`. On restart, the
spawn receives these same args.

## Implementation sketch

```zig
// ─── src/coding/restart.zig ───────────────────────────────────────
//! Self-restart via spawn-and-exit (Unix + macOS).
//!
//! Spawns a fresh instance of the running binary, then exits the
//! current process. Caller must close the listen socket before
//! calling `spawn` to avoid EADDRINUSE.

const std = @import("std");

pub const Error = error{
    SpawnFailed,
} || std.mem.Allocator.Error;

/// The path to the running executable, cached at bootstrap.
var cached_exe_path: ?[]const u8 = null;
/// Deep copy of argv, owned by this module.
var cached_argv: ?[]const []const u8 = null;

pub fn init(argv: []const []const u8, io: std.Io, allocator: std.mem.Allocator) !void {
    cached_exe_path = try std.process.executablePathAlloc(io, allocator);
    var argv_copy = try allocator.alloc([]const u8, argv.len);
    for (argv, 0..) |arg, i| {
        argv_copy[i] = try allocator.dupe(u8, arg);
    }
    cached_argv = argv_copy;
}

pub fn deinit(allocator: std.mem.Allocator) void {
    if (cached_argv) |argv| {
        for (argv) |arg| allocator.free(arg);
        allocator.free(argv);
    }
    if (cached_exe_path) |p| allocator.free(p);
}

/// Spawn a fresh instance and exit the current process.
/// Never returns on success.
pub fn spawnAndExit(io: std.Io) Error {
    const exe = cached_exe_path orelse return error.SpawnFailed;
    const argv = cached_argv orelse return error.SpawnFailed;

    // Build argv_owned: [exe_path, ...original_args[1..]]
    // We skip argv[0] (the original binary name/path) and use
    // the resolved exe path instead.
    const args_count = argv.len; // includes original argv[0]
    const argv_for_child = try std.mem.Allocator.dupe(
        std.heap.page_allocator,
        []const u8,
        argv,
    );
    // argv_for_child[0] = exe;  // resolved path, more robust

    var child = std.process.spawn(io, .{
        .argv = argv_for_child,
        .stdin = .inherit,
        .stdout = .inherit,
        .stderr = .inherit,
    }) catch return error.SpawnFailed;
    child.detach();

    // Brief head start so the child can bind before we exit.
    // 200ms is generous — in practice the child calls bind within ~1ms.
    std.time.sleep(200 * std.time.ns_per_ms);

    std.process.exit(0);
}
```

### Integration points

1. **Proxy mode** (`src/coding/modes/proxy.zig`):
   - In `initSession()`, call `restart.init(cfg.original_argv, io, allocator)`
     to cache argv + exe path.
   - Add `restart_requested: std.atomic.Value(bool)` to `Session`.
   - After the accept loop exits, check the flag:
     ```zig
     if (session.restart_requested.load(.acquire)) {
         server.deinit(io);   // close listen socket → port freed
         restart.spawnAndExit(io);  // spawn + exit
     }
     ```

     SSE frame, and the main loop picks it up.
   - Add `POST /restart` HTTP route for external callers.

   - v1 not supported. The mode is single-shot — no long-lived process.
