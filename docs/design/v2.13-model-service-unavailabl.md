## Goal

Let the agent loop backoff for a while and then retry the request. The agent should be able to handle this gracefully without crashing or getting stuck in an infinite loop. The user should also be informed about the issue and the retry attempts.

Also we could define a model priority in settings.json or a fallback model provide to use when the primary model provider is unavailable. This way, the agent can switch to a different model if the primary one is overloaded.
