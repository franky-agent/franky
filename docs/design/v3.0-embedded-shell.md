# Embedded shell options for Zig integration

This note captures the findings from a quick survey of shell engines and parsers that could be embedded into a Zig program for policy enforcement.

## Requirement

The target use case is:

- near-`sh` / POSIX shell compatibility
- AST or execution interception
- ability to block commands before they run
- ideally embeddable from Zig

Example policy:

- allow `echo hello world`
- reject any `rm` command
- optionally reject the whole script if a forbidden command appears anywhere in the AST

## Important correction for the example script

The following script is **not valid POSIX sh** as written:

```sh
echo hello world
for i in /tmp
  rm -rfv i
eof
echo Bye
```

Issues:

- `for` needs `do ... done`
- `eof` should be `done`
- `rm -rfv i` uses the literal string `i`; in a shell loop it would usually be `"$i"`

A corrected version would be:

```sh
echo hello world
for i in /tmp; do
  rm -rfv "$i"
done
echo Bye
```

## Findings

### 1. `epsh`

- Embeddable POSIX shell implemented in Rust
- Focuses on non-interactive/script execution
- Explicitly mentions **parse-then-execute** and inspecting the AST before execution
- Has an **external command handler** for custom spawning / sandboxing
- This makes it the strongest candidate for a Zig program that wants to inspect and block shell commands

### 2. `mvdan/sh`

- Go-based shell parser / formatter / interpreter
- Strong POSIX shell focus
- Provides interpreter hooks that can replace command execution
- Good candidate if you want a library-style shell engine with policy checks

### 3. `brush`

- Bash/POSIX-compatible shell in Rust
- Has experimental zsh-style `preexec` / `precmd` hooks
- Useful for interactive shell behavior and some command interception
- Hooks are still experimental, so it is less attractive for a strict embedding/policy layer than `epsh`

### 4. Zig-native parsers

- A true Zig-native POSIX shell parser/interpreter does **not** appear to be mature yet
- I found `shlex`, but it is only tokenization, not a full shell AST parser
- I also found shell projects in Zig, but they are not yet a mature embedded POSIX shell library for production use

## Recommended architecture

For Zig, the practical design is:

1. parse shell text in an embeddable engine
2. inspect the AST in Zig or in a small FFI wrapper
3. reject the script if a forbidden command is found
4. otherwise execute it

For this project, the safest policy is usually to **reject the whole script** if any forbidden command appears.

## Example policy

Allow:

```sh
echo hello world
```

Reject:

```sh
rm -rf /tmp
```

Reject whole script:

```sh
echo hello
for i in /tmp; do
  rm -rfv "$i"
done
echo Bye
```

## Practical recommendation

If you need this today, the best candidates are:

1. **epsh** — best fit for AST inspection and blocking before execution
2. **mvdan/sh** — strong shell parsing/interpreter option
3. **brush** — promising, but hooks are experimental and it is more shell-oriented

For a Zig program, a likely implementation path is a small **Rust or Go C-ABI wrapper** around the shell engine, with Zig making the allow/deny decision.
