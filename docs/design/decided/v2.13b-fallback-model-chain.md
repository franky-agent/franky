# v2.13b — Fallback Model Chain

**Status:** decided — ready for implementation
**Depends on:** [v2.13 Phases 1+2](../../archive/design/v2.13-model-service-unavailabl.md) (implemented — `provider_retry` event, tunable budget)
**Affects:** `src/coding/settings.zig`, `src/agent/loop.zig`, `src/ai/retry.zig`, `src/ai/stream.zig`

---

## 1. Problem

When a provider's retry budget (Phase 2) exhausts, franky surfaces a terminal error. For
sessions with multiple viable providers configured, the user expects the agent to silently
try the next one rather than fail the turn.

---

## 2. Design

### Settings additions

```json
{
  "model": "claude-sonnet-4-6",
  "fallback_models": [
    "claude-haiku-4-5",
    "gemini-2.5-pro"
  ],
  "fallback_scope": "session"
}
```

**`fallback_models`** — ordered list of model identifiers to try after the primary exhausts
its Phase-2 retry budget. Each entry follows the same format as the top-level `model` field.
Opt-in only; no OOTB defaults.

**`fallback_scope`:**

| Value | Behaviour |
|---|---|
| `"turn"` | Fall back for this turn only; next turn retries the primary |
| `"session"` | Once switched, stay on the fallback for the rest of the session |

`"turn"` is best when the outage is transient and auto-recovery is desired.
`"session"` avoids flapping when the primary is unstable.

### Switching trigger

When Phase-2 retry budget on the current model exhausts → instead of surfacing terminal,
walk `fallback_models` in order, skipping any model whose credential is absent at startup.
Each fallback gets its own Phase-2 retry budget (bounded by the same `retry_max_total_ms`).

### Startup credential check

At session start, franky checks which models in `settings.json` have credentials configured
(not validated — just present) and prints:

```
Available models: claude-sonnet-4-6, gemini-2.5-pro  [claude-haiku-4-5: no credentials]
```

Models without credentials are excluded from the fallback walk for that session. Full
credential lifecycle (rotation, re-check mid-session) is deferred to v3.

### Stream events

Extend `provider_retry` (Phase 1) with a `using_fallback` field:

```zig
provider_retry: struct {
    attempt: u32,
    max_attempts: u32,
    delay_ms: u32,
    reason: errors.Code,
    provider_code: ?[]const u8 = null,
    provider_message: ?[]const u8 = null,
    http_status: ?u16 = null,
    using_fallback: ?[]const u8 = null,  // model id if switched, null if still on primary
},
```

Emit a one-time `agent_warning` event when a fallback model is activated:

```zig
agent_warning: struct {
    code: []const u8,             // "using_fallback"
    primary_model: []const u8,
    fallback_model: []const u8,
},
```

This surfaces the switch loudly in the TUI, web UI, and sibling app channels.

## 3. Cross-provider concerns

| Concern | Approach |
|---|---|
| **Auth** | Credential-present check at startup; absent models skipped; full lifecycle deferred to v3 |
| **Tool-call shape** | Already abstracted via the registry — translates per-provider; needs round-trip test |
| **Thinking blocks** | Transcript carries thinking blocks as `Message.content[]`; fallback model that doesn't support thinking ignores them on send but transcript stays intact |
| **Cost asymmetry** | `using_fallback` field on `provider_retry` + `agent_warning` event surfaces the switch; user/sibling decides |
| **Quality degradation** | `agent_warning{using_fallback}` makes the switch loud; no automatic quality assessment |
| **Automatic switch-back** | `fallback_scope: turn` recovers on next turn; `fallback_scope: session` stays on fallback; auto-promotion (primary healthy again) deferred to a later release |

---

## 4. Decisions

| Q | Decision |
|---|---|
| Q3 — OOTB fallback chains | None. Phase 3 is opt-in only via explicit `fallback_models` config |

---

## 5. Acceptance criteria

- `model: claude-sonnet-4-6` + `fallback_models: [gemini-2.5-pro]` + stub that simulates
  Anthropic 503 storm → agent completes the run via Gemini.
- Transcript records which provider answered each turn.
- `agent_warning{using_fallback}` event emitted exactly once at switch time.
- `fallback_scope: turn` → next turn retries Anthropic first.
- `fallback_scope: session` → all subsequent turns use Gemini.
- Model without credentials in `fallback_models` is silently skipped; next model in list tried.

---

## 6. Out of scope

- API key management, rotation, re-check mid-session (v3).
- Automatic switch-back to primary mid-session after `fallback_scope: session`.
- Per-tool fallback.
- Provider health dashboard in web UI.
