## 1. Problem

An automated deep-code audit of every module in the source tree (2026-07-03) surfaced **nine issues** at 🔴 critical and 🟡 high/medium severity. They fall into three families:

1. **Reliability bugs** — proxy subscriber leaks that exhaust connection slots, silent data loss from object-store sweep, brittle array splicing in compaction that can silently corrupt transcripts.
2. **Error-handling bugs** — `catch return` patterns that void `errdefer` chains, manual field-by-field deinit that leaks when the struct grows, `nowMillis` returning 0 and silently degrading timing guards.
3. **Test-coverage gaps** — zero integration tests for the four run-mode drivers, zero dedicated tests for the partial-JSON parser, missing completeness assertions on object-store sweep.

None of these are speculative. The proxy subscriber leak and the grep `\b` issue have both been reproduced in the wild and documented in `docs/todos/todos.md`. The remaining issues were confirmed by reading the source.

This doc groups the fixes into implementation waves. Each wave is self-contained and independently mergeable.

## 2. What already exists

### 2.1 Proxy subscriber management (`src/agent/proxy.zig`)

The proxy mode's SSE fan-out works as follows:

- Each connected HTTP client creates a `Subscriber` struct (lines ~350-380 of proxy.zig).
- The fan-out loop (`fanOutLocked`) broadcasts events to all subscribers; on write failure it sets `sub.closed = true` but does **not** shut down the read side of the socket.
- Each subscriber's reader thread is in a `while (!sub.closed.load(.acquire)) { readVec … }` loop (around line 1772) to detect client disconnect. The thread blocks on `readVec` waiting for data or FIN.
- If the client disconnects without sending FIN/RST (browser background tab, OS-level half-close, network blip), the `readVec` never returns, `closed = true` set by `fanOutLocked` is never observed, the `defer removeSub` never runs.
- After ~32 cycles (`max_subs = 32`) all slots are exhausted and no new connections can be established until process restart.

### 2.2 Object store (`src/coding/object_store.zig`)

Content-addressed blobs ≥32 KiB are spilled to `objects/<hex[0..2]>/<hex[2..]>`. The `sweep(allocator, io, root_path, keep)` function (lines 102-115):

1. Walks the entire object store directory tree.
2. For each file, reconstructs the full 64-char hex hash from the shard path.
3. Checks membership against the supplied `keep` list.
4. Any hash not in `keep` is deleted.

The `keep` list is assembled by the caller (typically `session.zig`'s transcript serialization) by recursing through every `Ref` in the transcript. If the caller misses a ref — for instance, when a content block was spilled during a previous write but the transcript being written references a different set of blocks — the orphaned blob is deleted and subsequent reads of any message that referenced it fail with `error.ObjectNotFound`.

### 2.3 Grep tool regex engine (`src/coding/tools/grep.zig`)

The grep tool uses Zig's regex engine with `std.mem.replacementSizeForRegex` / `regex: regexMod`. Zig's regex is an ECMAScript subset. Critically, **ECMAScript does not support `\b` (word boundary)**.

Models reliably emit `\b` in patterns (e.g. `fn grep|const grep|\.name.*=.*\"grep\"\b`) because every regex engine they know (PCRE, Python `re`, JavaScript, Ruby, Go) supports it. The grep tool returns `grep_bad_regex` with the suggestion "use regex=false for literal substring search", which is not actionable for regex-savvy models that actually need word boundaries.

The bug is documented with reproduction cases in `docs/todos/todos.md`.

### 2.4 `catch return` voiding `errdefer` (`agent/loop.zig`, `agent/agent.zig`)

Zig semantics: `errdefer` fires only when the *current function* returns an error via the error-union path (`try` or `return error`). A `catch return` in a void/non-error-returning function does **not** trigger `errdefer` — the function returns normally (via a `return` statement), not via the error path.

The following pattern appears in the agent code:

```zig
const team_owned = a.dupe(u8, team) catch return;  // returns normally
errdefer a.free(team_owned);                         // dead — never fires
const channel_owned = a.dupe(u8, channel) catch return;
errdefer a.free(channel_owned);                      // also dead
```

After `catch return`, the `errdefer` on the next line is dead code because the function already returned on the error path — or returned normally on success, in which case `errdefer` is never meant to fire. The pattern is correct only when the function's return type is `!void` (so `catch return` actually returns an error). When used inside a void helper called from `!void`, or when `catch return` is the early-return path and a subsequent `try` is the error path, the `errdefer` is never reached.

### 2.5 `freeSessionHeader` fragile (`src/coding/session.zig:136-146`)

`SessionHeader` has 7 owning `[]const u8` fields. `freeSessionHeader` manually frees each one by name:

```zig
pub fn freeSessionHeader(allocator: std.mem.Allocator, h: SessionHeader) void {
    allocator.free(h.id);
    allocator.free(h.title);
    // … 5 more lines …
    allocator.free(h.system_prompt_hash);
}
```

Adding a new owning-string field to `SessionHeader` (e.g. `metadata`, `session_group`) silently leaks unless `freeSessionHeader` is also updated. There is no compile-time check.

### 2.6 Mode-level integration tests missing

The four run-mode drivers (`print.zig`, `interactive.zig`, `rpc.zig`, `proxy.zig`) have zero integration tests. The CLI argument parser is tested extensively, but the mode-entry functions themselves — which assemble the `Agent`, `Registry`, permission state, and session from CLI flags — are never exercised end-to-end.

The `kitchen_sink_test.zig` tests the RPC framer (raw message serialization) but does not test the full `--mode rpc` lifecycle (handshake → request → response → close).

### 2.7 `nowMillis()` returns 0 on error (`src/ai/stream.zig:36-53`)

`nowMillis()` tries two code paths depending on the OS:

1. Linux: `clock_gettime(CLOCK_REALTIME, …)` direct syscall.
2. Other POSIX (Darwin/BSD): libc `clock_gettime(CLOCK_REALTIME, …)`.

Both return 0 on any failure. Callers in the agent loop, SSE handler-stall detection, and the TUI's usage timing all use `nowMillis()` to compute elapsed time. When `nowMillis()` returns 0, elapsed-time calculations produce garbage (negative or epoch-sized values), which can cause the stuck detector to miss stalls or the SSE timeout to fire prematurely.

In practice only the libc path (macOS/BSD) can fail, and only under rare conditions (misconfigured system clock, sandbox restrictions on `clock_gettime`). But the fallback-to-0 pattern means "clock unavailable" is indistinguishable from "epoch zero".

### 2.8 Compaction `replaceSpanWithSummary` uses manual `memcpy` (`src/coding/compaction.zig:556-566`)

The compaction module's `replaceSpanWithSummary` function removes a span of messages from the transcript `ArrayList` and inserts the summary message in their place. It implements the splice manually with `std.mem.copyForwards` rather than using the `ArrayList`'s built-in `replaceRange` (which doesn't exist in the current stdlib surface).

The manual copy is correct for the current call sites but is brittle — a single off‑by‑one error on the source or destination region corrupts the transcript silently. There is no invariant check or assertion after the copy.

### 2.9 Partial JSON parser (`src/ai/partial_json.zig`)

The incremental JSON parser is a complex state machine that reconstructs tool-call arguments from streaming deltas. It handles nested objects, strings with escape sequences, arrays, and partial tokens at chunk boundaries.

Despite being critical for correct streaming tool-call parsing — and having been the site of past bugs (todos.md documents DeepSeek-specific parse failures) — it has **no dedicated unit tests**. Coverage is entirely indirect through provider integration tests.

## 3. Proposed fixes

### 3.1 [🔴] Fix proxy subscriber slot leak — Wave 1

**Target:** `src/agent/proxy.zig`

**Approach:** When `fanOutLocked` sets `sub.closed = true` (because the write failed or the fan-out loop is shutting down), also shut down the **read side** of the subscriber's TCP socket. This forces the reader thread's `readVec` to return immediately (with `ConnectionReset` or EOF), the `break` in the read loop fires, and the `defer removeSub` runs.

```zig
// Inside fanOutLocked, after setting sub.closed = true:
if (sub.shutdown_on_close) {
    // Shutdown the read side so the reader thread unblocks.
    std.posix.shutdown(sub.fd, .RECV) catch {};
}
```

Add a `shutdown_on_close: bool` field to `Subscriber`. Set it to `true` for all normal SSE subscribers; set it to `false` for any internal subscribers that share a file descriptor with the listen socket (none exist today, but the flag prevents accidentally shutting down the wrong fd).

**Edge cases:**

- `shutdown` may fail if the fd is already closed or invalid — this is non-fatal; the `catch {}` is correct.
- The reader thread may observe EOF before `removeSub` runs — this is fine; the `break` in the read loop handles both `n == 0` (clean FIN) and error (RST).
- On macOS, `std.posix.shutdown` with `.RECV` is `shutdown(fd, SHUT_RD)`. This is POSIX-standard and works identically.

**Testing:** Add a proxy-mode integration test that:
1. Opens an SSE connection to `http://127.0.0.1:8787/events`.
2. Reads one event.
3. Closes the TCP socket without a graceful HTTP/2 FIN (just `std.posix.shutdown(fd, .SEND)` or drop the `std.Io.File`).
4. Opens a second SSE connection.
5. Asserts the second connection receives events without `subscriber.refused`.

**LOC:** ~10 lines in proxy.zig + ~50 lines test.

### 3.2 [🔴] Harden object store `sweep` against incomplete keep lists — Wave 1

**Target:** `src/coding/object_store.zig`, `src/coding/session.zig`

**Approach A (assertion gate — recommended):** After assembling the `keep` list and before calling `sweep`, re-read every `Ref` in the transcript and assert that its hash is present in `keep`. This makes the invariant explicit and catches call-site bugs at runtime in debug/release-safe builds.

```zig
// In session.zig, before calling object_store.sweep:
if (builtin.mode != .ReleaseFast) {
    for (transcript.messages.items) |msg| {
        for (msg.content) |cb| {
            if (cb == .ref) {
                const hash = cb.ref.hash;
                std.debug.assert(std.mem.indexOfScalar([]const u8, keep, hash) != null);
            }
        }
    }
}
```

**Approach B (defensive):** Change `sweep` to take the transcript directly instead of a `keep` list, so it extracts refs itself and there is no caller-side risk. This is cleaner but couples `object_store` to `agent.loop.Transcript`, which it currently doesn't depend on.

**Recommendation:** Approach A for Wave 1 (minimal coupling, immediate safety), defer Approach B to a later refactor if the coupling proves painful.

**Testing:** Extend `kitchen_sink_test.zig` or add a dedicated `object_store_test.zig` that:
1. Writes several objects to the store.
2. Writes a transcript that references only some of them.
3. Runs `sweep` with an intentionally incomplete keep list (one hash missing).
4. Asserts the assertion catches it (in Debug/ReleaseSafe) or that the missing ref object is still readable (in ReleaseFast the assertion is stripped, but the existing sweep logic should not delete the ref'd object because the keep list is correct in production).

**LOC:** ~10 lines in session.zig + ~60 lines test.

### 3.3 [🔴] Handle grep regex `\b` gracefully — Wave 2

**Target:** `src/coding/tools/grep.zig`

**Three approaches, in priority order:**

**Approach A (strip known-unsupported patterns, recommended):** Before passing the model-supplied pattern to the regex engine, scan for `\b`, `\B`, `(?=`, `(?!`, `(?<=`, `(?<!` — assertions that ECMAScript does not support — and strip them (replace `\b` with empty string, replace `\B` with empty string, handle lookaround by warning). This is a best-effort transformation: it may change semantics, but the result is *more useful* than returning an error.

The replacement function:
1. Walks the pattern string.
2. When it encounters `\b`, removes the two bytes.
3. When it encounters `\B`, removes the two bytes.
4. When it encounters `(?=` or similar, returns `error.UnsupportedRegexAssertion` with a `use regex=false` hint (these can't be stripped without semantic change).

```zig
fn stripUnsupportedAssertions(pattern: []const u8, allocator: std.mem.Allocator) ![]u8 {
    var out: std.ArrayList(u8) = .empty;
    errdefer out.deinit(allocator);
    var i: usize = 0;
    while (i < pattern.len) {
        if (pattern[i] == '\\' and i + 1 < pattern.len) {
            if (pattern[i + 1] == 'b' or pattern[i + 1] == 'B') {
                i += 2;
                continue;
            }
        }
        if (i + 3 < pattern.len and std.mem.eql(u8, pattern[i..i+3], "(?=")) return error.UnsupportedRegexAssertion;
        // handle (?!, (?<=, (?<!
        try out.append(allocator, pattern[i]);
        i += 1;
    }
    return out.toOwnedSlice(allocator);
}
```

**Approach B (ECMAScript fallback mode):** When the regex fails to compile, automatically retry with `regex = false` (literal substring search) and append a warning to the result. This is less invasive than stripping but may produce false positives (literal matches when the model intended a word-boundary match).

**Approach C (switch regex engine):** Replace Zig's built-in regex with a PCRE2 binding or a Zig-native engine that supports `\b`. High complexity and dependencies — deferred to post-1.0.

**Recommendation:** Approach A for Wave 2, Approach B as a fallback for the remaining unsupported assertions. If `stripUnsupportedAssertions` returns `error.UnsupportedRegexAssertion`, fall through to literal-substring mode.

**Testing:** Add tests for `stripUnsupportedAssertions` covering `\b`, `\B`, `(?=`, plain patterns, and patterns with `\\` (literal backslash before b).

**LOC:** ~40 lines in grep.zig + ~30 lines test.

### 3.4 [🟡] Fix `catch return` voiding `errdefer` — Wave 2

**Target:** `src/agent/loop.zig`, `src/agent/agent.zig`

**Approach:** Audit every instance of `catch return` followed by `errdefer` referencing the caught value. The fix depends on the context:

- **If the containing function returns `!void`:** Replace `catch return` with `try`. The `errdefer` fires on the `try` error path. This is the simplest and most common fix.

  ```zig
  // Before:
  const team_owned = a.dupe(u8, team) catch return;
  errdefer a.free(team_owned);
  const channel_owned = a.dupe(u8, channel) catch return;
  errdefer a.free(channel_owned);

  // After:
  const team_owned = try a.dupe(u8, team);
  errdefer a.free(team_owned);
  const channel_owned = try a.dupe(u8, channel);
  errdefer a.free(channel_owned);
  ```

- **If the containing function returns `void`:** Wrap the body in a `!void` helper that returns errors, so `errdefer` has a meaningful error path:

  ```zig
  fn doStuff(a: Allocator) void {
      const x = mayFail() catch return;  // void fn — can't use try
      errdefer a.free(x);                // dead code
  }

  // After:
  fn doStuff(a: Allocator) void {
      doStuffHelper(a) catch {};
  }
  fn doStuffHelper(a: Allocator) !void {
      const x = try mayFail();
      errdefer a.free(x);
      // … body …
  }
  ```

- **If the `catch return` is in a `for` loop or block:** Same treatment — extract to a `!void` helper or restructure to use `try`.

**Search pattern to audit:**
1. Find every `catch return` in agent/loop.zig and agent/agent.zig.
2. Check if the next statement (or any statement before the next `try`) is an `errdefer` whose argument depends on the caught value.
3. Apply the appropriate fix above.

**Testing:** The existing agent-loop tests (`agent_loop_test.zig`, `agent_class_test.zig`) exercise the code paths that currently have this bug. After the fix, run them with `std.testing.allocator` — the allocator will detect any leaked memory that the dead `errdefer` was supposed to free.

If the existing tests don't cover the specific code path, add a test that triggers the error path (e.g. simulate an OOM by wrapping the allocator).

**LOC:** ~10-30 lines changed across loop.zig and agent.zig.

### 3.5 [🟡] Make `freeSessionHeader` field-safe — Wave 2

**Target:** `src/coding/session.zig`

**Approach A (compile-time reflection — recommended):** Replace the manual field-by-field free with a `comptime` loop over the struct fields, freeing only those that are `[]const u8`:

```zig
pub fn freeSessionHeader(allocator: std.mem.Allocator, h: SessionHeader) void {
    inline for (@typeInfo(SessionHeader).Struct.fields) |field| {
        if (field.type == []const u8) {
            allocator.free(@field(h, field.name));
        }
    }
}
```

This automatically handles any new `[]const u8` field added to the struct. Non-string fields (timestamps, enums, optionals) are skipped.

**Alternative (approach B):** Add a `comptime` assertion that `freeSessionHeader` frees every string field. This is more explicit but more boilerplate.

**Edge cases:** `?[]const u8` (optional string) fields are not handled by the `field.type == []const u8` check — they have type `?[]const u8`. If such a field is added, the deinit must be either explicit (non-reflective) or the check must handle optionals. Since `SessionHeader` currently has no optional string fields and none are planned, this is a future concern.

**Testing:** Add a test that creates a `SessionHeader`, frees it with `freeSessionHeader`, and asserts no memory leak (implicit — `std.testing.allocator` catches it).

**LOC:** ~3 lines changed in session.zig.

### 3.6 [🟡] Add mode-level integration tests — Wave 3

**Target:** New file `test/mode_print_test.zig` (and siblings for rpc/proxy)

**Approach:** For each mode, write a minimal integration test that:

### 3.7 [🟡] Improve `nowMillis()` fallback — Wave 3

**Target:** `src/ai/stream.zig`

**Approach A (recommended):** Add a `mach_absolute_time` fallback on Darwin platforms before returning 0:

```zig
pub fn nowMillis() i64 {
    if (builtin.os.tag == .linux) {
        // … existing Linux path …
    }
    // Darwin: use mach_absolute_time which cannot fail.
    if (builtin.os.tag == .macos or builtin.os.tag == .ios) {
        const mt = std.c.mach_absolute_time();
        // Convert from mach timebase to milliseconds.
        var info: std.c.mach_timebase_info_data_t = undefined;
        _ = std.c.mach_timebase_info(&info);
        const nanos = mt * @as(u64, info.numer) / @as(u64, info.denom);
        return @as(i64, @intCast(nanos / std.time.ns_per_ms));
    }
    // POSIX via libc (existing fallback — keep as-is).
    if (builtin.link_libc) {
        var ts: std.c.timespec = undefined;
        if (std.c.clock_gettime(.REALTIME, &ts) == 0) {
            return @as(i64, ts.sec) * 1000 + @divFloor(@as(i64, ts.nsec), std.time.ns_per_ms);
        }
    }
    return 0;
}
```

**Approach B (simpler):** Change the return type to `?i64` so callers must handle `null` explicitly. This is semantically cleaner but requires changes at every call site (10-15 locations). Deferred.

**Recommendation:** Approach A for Wave 3 — low risk, no call-site changes, eliminates the Darwin edge case entirely.

**Testing:** Run the existing tests on macOS (the `nowMillis` tests are in `stream.zig` inline tests). If no macOS CI is available, the change is low-risk (Darwin-only, compile-time-gated) and the existing Linux CI covers the Linux path.

**LOC:** ~10 lines added in stream.zig.

## 4. Migration / rollback

Every fix in this doc is a forward change. None require data migration:

- **Proxy shutdown:** Changes runtime behavior of the fan-out loop. Rollback = revert the change.
- **Object store sweep assertion:** `std.debug.assert` is stripped in ReleaseFast. Rollback = revert or remove the assertion line.
- **Grep regex stripping:** Only activates when a `\b`/`\B` pattern is encountered. Rollback = revert the strip function.
- **`catch return` → `try`:** Local change; rollback = revert the affected lines.
- **`freeSessionHeader` reflection:** Same observable behavior; rollback = restore manual free lines.
- **Mode tests:** New files. Rollback = delete them.
- **`nowMillis` Darwin path:** Compile-time-gated. Rollback = revert the added block.
- **Compaction assertion:** Stripped in ReleaseFast. Rollback = revert.
- **Partial JSON tests:** New file. Rollback = delete it.

No wire-format changes, no JSON-schema changes, no breaking API changes.

## 6. Open questions

### Q6.1 — Proxy shutdown on macOS

**`std.posix.shutdown` with `.RECV` is POSIX-standard.** Does Zig's stdlib expose `SHUT_RD` under the expected enum variant on macOS?

**Answer:** Yes. `std.posix.ShutdownHow` has `.RECV`, `.SEND`, and `.BOTH`. The underlying system call is `shutdown(fd, how)` where `how` is an `int` (0=RD, 1=WR, 2=RDWR). Zig's enum maps to these values. Verified in `std.posix.shutdown` for Linux and macOS.

### Q6.2 — Object store sweep assertion in ReleaseFast

`std.debug.assert` is stripped in ReleaseFast and ReleaseSmall. Should we use a `builtin.mode` check with `@panic` instead?

**Proposed:** Keep `std.debug.assert`. ReleaseFast is already "trust the programmer" mode. The assertion is a development aid, not a security boundary. If we want the check in production, we can promote it to an explicit `if` + `log.err` later.

**Decision:** ✓ accept — `std.debug.assert` is sufficient.

### Q6.3 — Mode integration test binary path

How does the test binary find the `franky` binary to spawn as a subprocess?

**Proposed:** `build.zig` passes the exe path as a compile-time constant (a `pub const` in a generated module) or the test discovers it relative to `argv[0]`. The simpler approach: the test assumes `../zig-out/bin/franky` relative to the cwd (which is the repo root when `zig build test` runs). Add a `comptime` check that the file exists and emit a clear error if it doesn't.
